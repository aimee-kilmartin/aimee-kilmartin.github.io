How does Javascript work with HTML and CSS?

Javascript works with HTML and CSS like the functionality of a house. The blueprint of the house is created by HTML - a template on the structure of each piece of the building frame. CSS is the styling that determines what each of those pieces look like - selecting a colour on the walls, thickness of the carpet, what style of light fittings etc..

Then Javascript comes along, and turns on those lights, brings water through the kitchen tap - hot or cold - and when you push the button the garage door opens.

Javascript makes the items built by HTML and styled by CSS a functional interactive piece.

What is the DOM and how does it fit in with Javascript?

The DOM stands for Document Object Model. It is a programming interface which translates programming languages such as HTML and CSS into objects so that other languages such as JAvascript can interpret it

The DOM is structured like a family tree - HTML trees work from top to bottom so the top of the tree is the root of the document (HTML element), and then branches are different parent elements, followed by one or more child elements. Each child element can then be a parent to a different element and so on. Each element can only have one parent element however a parent can have multiple children within the same hierarchy.

  • The document is the name for the web page
  • The object is the content of the webpage - content being the broad term including words, images, elements such as divs, attributes, styles etc. All the pieces of the puzzle
  • The model is the representation of how these are put together. It is the picture on the box to be able to put the puzzle together, and instructions on how to do so

The DOM can be interacted with directly on a webpage in the browser - this will change the local version of the site. It can also be interacted with from the code editor - whilst writing Javascript, it is a good habit to use the command console.log(function) regularly.

This will run the function in the console log part of the DOM, outputting the result of the function. This is very helpful to ensure that functions and the code are working in the expected way.

What is Control Flow and how do Loops affect this?

Control Flow is the order in which the Javascript is processed. The general way in this works is from top to bottom - BUT there are many things that can interrupt this.

The control flow is like emptying a dishwasher.

The majority of the Javascript runs as such:

  • Open the dishwasher when it is clean
  • Take cups out, put in the cupboard
  • Take large bowls out, stack, and put in the cupboard
  • Take small plates out, stack, and put in the cupboard

And so on and so on..

Then you come across a loop - in Javascript this is literally called a loop, but also if I find a dirty plate amongst the clean load of dishes, it throws me for a loop! The take small plates out step is now interrupted because two small plates are DIRTY. If the item is clean it is put away. Else ALERT!! DIRTY ITEM!!

Javascript interpretation of the loop:

  1. The loop starts with the counter at 0 dishes washed
  2. It will continue looping until the counter of dirty dishes washed is equal to the number or dirty dishes there were to begin with (2)
  3. After each loop the counter will increase by one to record that loop as complete

The instructions of what to do during the loop are processed between steps 2 and 3, but are written below the loop (against the standard control flow in itself!):

  1. Put the dish in the sink
  2. Turn on the tap to hot water at a medium pressure
  3. Add dish soap to the scrubber
  4. Scrub scrub scrub
  5. Rinse the dish
  6. Place dish in the rack to dry off

Is there another dirty dish? Yes, repeat the loop

  1. No, turn off the tap and put down the scrubber
  2. Dry hands

End of the loop: carry on unloading clean dishes from the dishwasher

What is the Difference between Accessing Data from Arrays and Objects?

Arrays and Objects are structured differently, and therefore need to be accessed differently. First of all; what are arrays and what are objects?

Arrays are a list of Elements (items) - this is commonly used for number values, but can also be a list of things with just one data point (ie ‘Banana’, with no other information needed). Each element in the array will be assigned a value from left to right, starting with 0.

Let freshFood = [Banana, Broccoli] [0] [1]

Objects are similar to arrays however they can have multiple data points. Each object will contain one or more property (also known as ‘key pairs’) which have a key name and a value

Let banana = { Colour: Yellow Type: Fruit Price: 1.50 } Let broccoli = { Colour: Green Type: Vegetable Price: 2.50 }

Accessing the data:

As the elements within an array are defined by their index number, this is what is used to call the data. So using console.log(freshFood[1]) will call the element Broccoli into the console

The properties in an object are accessed using object.key in a function. So console.log(banana.type will result in “an output of Fruit”

What are Functions and why are they useful?

Functions are a group of actions that are stored within a name created by the programmer.Then this name of the function can be used to call (run) the function at any point in the script, or they can be used in other functions later on by inputting the name of the function, rather than repeating the whole set of actions.

The benefits of this are consistency, changeability, readability

Consistency

Each time a function is called it will be called the same> as it is running off the same piece of code. This also reduces errors as the same groups of code aren’t typed out repeatedly throughout the Javascript file

Changeability

If a function is to run X of Y, and this needs to be updated to X of Z in the future, this is easily done in the one original function. Because the future mentions of this function all run from the original one, they don’t need to be updated

Readability

Calling a function by a name eg makeXintoY is a short span of text, rather than inputting a block of code into other blocks of code. This not only makes the code shorter and more concise, but someone looking at the code will know what that piece of a function does (roughly) by it’s name and be able to search for other instances of that function being called within the script
